Diagrama de Proceso: Gestión de Configuraciones de Enrutadores

Un enfoque visual de la recolección, almacenamiento y extracción de datos

Este diagrama ilustra las actividades clave y la secuencia lógica en la gestión de configuraciones de enrutadores. Se divide en tres fases principales que cubren desde la obtención de la información hasta su presentación al usuario, asegurando un flujo de trabajo eficiente y controlado.

1. Recolección de Datos

Esta fase se centra en la obtención segura de la información de configuración directamente desde los enrutadores.

1.1

Inicio del Proceso de Recolección

El sistema inicia la tarea de recolectar configuraciones, ya sea de forma programada o manual.

↓
1.2

Conexión con Enrutador

El sistema establece una conexión segura con cada enrutador objetivo (ej., vía SSH, Telnet, SNMP).

↓
1.3

Autenticación

El sistema se autentica en el enrutador utilizando credenciales predefinidas.

↓
1.4

Extracción de Configuración

Se ejecutan comandos específicos en el enrutador para extraer su configuración actual.

↓
1.5

Verificación de Datos Recolectados

Se realiza una validación básica para asegurar que los datos son completos y válidos.

↓

Si 'No', se registra un error y notifica la falla.

2. Almacenamiento de Datos

Una vez recolectados y validados, los datos de configuración se almacenan de forma segura y accesible.

2.1

Formateo y Normalización de Datos

La configuración extraída se procesa y se formatea en un esquema estandarizado.

↓
2.2

Generación de Metadatos

Se adjuntan metadatos relevantes (fecha, ID del enrutador, versión de firmware) a la configuración.

↓
2.3

Almacenamiento en Base de Datos

Los datos formateados y los metadatos se guardan en la base de datos central.

↓
¿Almacenamiento exitoso?
Sí → No →
↓

Si 'No', se registra un error de almacenamiento y notifica.

↓
2.4

Indexación (Opcional)

Los datos pueden ser indexados para optimizar futuras búsquedas y extracciones.

3. Interfaz de Usuario para Extracción de Configuraciones

Esta fase describe cómo un usuario interactúa con el sistema para acceder a las configuraciones almacenadas.

3.1

Acceso a la Interfaz de Usuario

El usuario inicia sesión o accede a la aplicación web/cliente.

↓
3.2

Selección de Criterios de Búsqueda

El usuario especifica los parámetros para buscar configuraciones (ej., nombre del enrutador, rango de fechas, palabras clave).

↓
3.3

Envío de Solicitud de Extracción

La interfaz de usuario envía la solicitud de búsqueda a la capa de lógica de negocio/API.

↓
3.4

Consulta a la Base de Datos

El sistema consulta la base de datos utilizando los criterios proporcionados por el usuario.

↓
3.5

Presentación de Resultados

Las configuraciones coincidentes se recuperan y se presentan al usuario.

↓

↓
3.6

Opción de Exportación/Descarga

El usuario puede exportar o descargar las configuraciones en varios formatos.

↓
3.7

Fin de la Interacción del Usuario

El usuario cierra la sesión o finaliza la tarea de extracción.
